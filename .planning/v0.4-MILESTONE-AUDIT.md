### Gaps (3)

**INTG-02 — UNSATISFIED: Telegram deadline notifications never sent**

`TelegramSync.push_deadlines()` exists and is tested, but is never called from `main.py`. The `deadline_sync` table on bot server is always empty. The scheduler runs daily at 09:00 UTC but `get_alerts_for_user()` returns `[]` — no messages are ever sent.

Fix: Call `_sync.push_deadlines(alerts)` in `main.py` after computing attention panel alerts. Requires instantiating `TelegramSync` outside the queue-fetch guard.

**FUND-03 — PARTIAL: Provider abstraction bypassed**

`providers/` package fully built with `get_provider()` factory, but `controller.py` passes `provider=None` to `extract_metadata()`, triggering `_legacy_mode` in `ai_extractor.py`. Setting `config.active_provider = "openrouter"` has no effect.

Fix: Wire `get_provider(config)` call in `controller.py` before `extract_metadata()`, pass the provider instance. Remove `_legacy_mode` path.

**PROF-01 — PARTIAL: Auto-bind advisory-only**

`auto_bind_results()` computes fuzzy matches and displays summary in UI. `move_record_to_client()` exists in `controller.py` but is never called. Documents are not actually migrated between client DBs after auto-bind display.

Fix: Add confirmation button in `main.py` that calls `move_record_to_client()` for selected bindings.
