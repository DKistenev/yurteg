# Requirements: ЮрТэг v0.9 Backend Hardening

**Defined:** 2026-03-26
**Core Value:** Юрист загружает папку с документами и за 20 минут получает готовый реестр с метаданными — без ручного ввода, без обучения, без «проекта внедрения»

## v0.9 Requirements

### Cleanup

- [x] **CLEAN-01**: Удалить validator.py и все вызовы L1-L5 валидации из controller.py и тестов
- [x] **CLEAN-02**: Удалить reporter.py (Excel) и все вызовы из controller.py, UI и тестов
- [ ] **CLEAN-03**: Удалить deprecated `_create_client()`, `_try_model()` из ai_extractor.py
- [ ] **CLEAN-04**: Удалить thinking mode setting из UI и config (ZAI-specific)
- [ ] **CLEAN-05**: Вычистить мёртвый код облачной эры (unused imports, fallback chains)

### AI Pipeline

- [ ] **AI-01**: Доработать GBNF грамматику — убрать confidence, ужесточить даты, опционально contract_number
- [ ] **AI-02**: Реализовать confidence через logprobs Ollama (два запроса или heuristic) — помечать подозрительные доки для перепроверки
- [ ] **AI-03**: Починить постпроцессор — whitelist латинских аббревиатур (NDA, SLA, GPS, ИНН) в cyrillic_only профиле

### Redline

- [ ] **RED-01**: Реализовать word-level redline DOCX с track changes (w:ins/w:del) вместо sentence-level
- [ ] **RED-02**: Единый redline-движок — сравнение версий документа + сравнение с шаблоном → скачать .docx
- [ ] **RED-03**: Починить баг rPr в version_service.py (track changes теряют форматирование в Word)

### Vectors

- [ ] **VEC-01**: Кэширование embeddings шаблонов — миграция v8, таблица template_embeddings
- [ ] **VEC-02**: Пометить документ как шаблон из карточки (mark_contract_as_template → UI кнопка)
- [ ] **VEC-03**: Автоподбор шаблона для документа — векторный поиск (бэкенд есть, подключить к UI)
- [ ] **VEC-04**: Передавать полный текст документа в embedding (убрать truncation [:8000])

### UI Wiring

- [ ] **WIRE-01**: Открытие файла в нативном приложении (open/startfile/xdg-open) из карточки документа
- [ ] **WIRE-02**: Bulk deletion — реальный handler удаления документов из реестра
- [ ] **WIRE-03**: Панель дедлайнов — get_attention_required() → виджет в реестре
- [ ] **WIRE-04**: Кнопка «Сохранить как шаблон» в карточке документа

### Stability

- [x] **STAB-01**: atexit.register() вызывать 1 раз (не в retry-цикле llama-server)
- [x] **STAB-02**: Предупреждение пользователю при обрезке текста >30K символов

## Future Requirements

### Deferred

- **OCR**: Обработка сканированных PDF (Tesseract/Vision API) — не в рамках MVP
- **Telegram**: Полноценная интеграция бота (очередь, уведомления) — отдельная веха
- **DMG/EXE сборка**: Доставка конечным пользователям — после стабилизации бэкенда
- **Audit logging**: Журнал действий — нет value без мультиюзера
- **AI response cache**: Кэширование по file_hash — оптимизация, не блокер

## Out of Scope

| Feature | Reason |
|---------|--------|
| Починка облачных провайдеров (ZAI, OpenRouter) | Архивный код, всё строим под Ollama |
| Мультипользовательский доступ | Требует серверной архитектуры |
| Google Drive интеграция | OAuth complexity, не нужна для MVP |
| Docker-упаковка | DMG/EXE модель доставки |
| OCR | Не в рамках текущего MVP |
