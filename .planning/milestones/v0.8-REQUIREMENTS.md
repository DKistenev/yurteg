# Requirements: ЮрТэг v0.8 Hardening & Cleanup

**Defined:** 2026-03-25
**Core Value:** Юрист загружает папку с документами и за 20 минут получает готовый реестр — без ручного ввода, без обучения, без «проекта внедрения»

## v0.8 Requirements

### Целостность данных (DATA)

- [x] **DATA-01**: При повторной обработке документов заметки юриста и ручные статусы сохраняются
- [x] **DATA-02**: Связи между документами (версии, платежи) не рвутся при повторной обработке
- [x] **DATA-03**: Платёжные данные (сумма, частота, направление) сохраняются в базу после обработки
- [x] **DATA-04**: В поле «предмет» отображаются реальные имена, а не маски [ФИО_1]

### Исправление интерфейса (UIFIX)

- [x] **UIFIX-01**: Клик по строке реестра открывает боковую панель предпросмотра без ошибок
- [x] **UIFIX-02**: Кнопка «Скачать PDF» на карточке документа скачивает файл
- [x] **UIFIX-03**: Кнопка «Переобработать» запускает повторную обработку документа
- [x] **UIFIX-04**: Порог напоминаний в настройках сохраняется и применяется при перезапуске
- [x] **UIFIX-05**: Массовая смена статуса показывает читаемые названия, а не технические данные
- [x] **UIFIX-06**: Ошибки в интерфейсе записываются в лог, а не проглатываются молча

### Чистка кода (CLEAN)

- [x] **CLEAN-01**: Старый интерфейс (Streamlit, 1700 строк) удалён — одна точка входа
- [x] **CLEAN-02**: Неиспользуемые функции и дубликаты удалены из бэкенда
- [x] **CLEAN-03**: Падающие и отключённые тесты починены или удалены — все тесты зелёные

### Подготовка к релизу (PROD)

- [x] **PROD-01**: Тесты написаны для модулей без покрытия (сканер, извлечение текста, Excel-отчёт, постобработка, оркестратор)
- [x] **PROD-02**: Шрифты и календарь загружаются из приложения, не из интернета — работает офлайн
- [x] **PROD-03**: Все библиотеки указаны в requirements.txt с точными версиями — установка на другом компе не ломается
- [x] **PROD-04**: Локальная модель корректно находит сервер, проверка качества работает через провайдер-систему

## Deferred (v0.9+)

- Сборка DMG для macOS
- Сборка EXE для Windows
- Автообновление
- OCR для сканированных PDF

## Out of Scope

| Фича | Причина |
|-------|---------|
| is_template в базе | Решили не спрашивать AI про шаблоны |
| confidence в базе | Считаем через logprobs, не через поле модели |
| Unused CSS/tokens cleanup | Мелочь, не влияет на работу |
| Google Drive интеграция | Telegram покрывает сценарий |
| Совместный доступ | Требует серверной БД |
