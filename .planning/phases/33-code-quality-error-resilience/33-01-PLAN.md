# Phase 33 Plan 01: Code Quality & Error Resilience

## Objective
Устранить P1 технический долг (hardcoded colors, dead code, a11y, DRY violations) и добавить graceful degradation при сбоях бэкенда (loading states, error toasts).

## Context
- @33-CONTEXT.md — все решения из аудита
- @app/styles.py — design tokens (HEX dict, Tailwind constants)
- @app/static/tokens.css — CSS custom properties --yt-*

## Tasks

### Task 1: Extract shared date utilities (AUDIT-07)
type="auto"

Create `app/utils.py` with `_MONTHS_RU` and `_format_date_ru`. Update imports in:
- `app/pages/document.py` — remove local `_MONTHS_RU` + `_format_date_ru`, import from utils
- `app/components/split_panel.py` — remove local `_MONTHS_RU` + `_format_date_ru`, import from utils
- Note: registry.py does NOT have _MONTHS_RU/_format_date_ru (confirmed by grep)

**Done:** `_MONTHS_RU` and `_format_date_ru` exist only in `app/utils.py`.

### Task 2: Replace inline hardcoded hex with CSS vars (AUDIT-05)
type="auto"

Replace hardcoded hex in these files where matching --yt-* tokens exist:
- `app/pages/settings.py`: icon_bg/icon_color in summary cards, section header icon color
- `app/components/split_panel.py`: inline style colors (#1e293b, #94a3b8, #e2e8f0, #475569)

**Scope:** Only replace colors that have matching --yt-* tokens. Leave AG Grid JS preview (registry.py), SVG strokes, status badge colors, and TMPL_TYPE_COLORS as-is.

**Done:** No hardcoded hex in component inline styles where a --yt-* token exists.

### Task 3: Bulk actions a11y — ui.button instead of ui.html (AUDIT-06)
type="auto"

Replace `ui.html('<button...')` in `app/components/bulk_actions.py` with `ui.button()` for keyboard accessibility (Tab + Enter).

**Done:** Bulk action buttons are proper `ui.button()` elements with `aria-label`.

### Task 4: Settings summary cards scroll-to-section (AUDIT-04)
type="auto"

Implement scroll-to-section via JS `scrollIntoView` when clicking summary cards in settings.py. Add `id` attributes to section headers, wire card click to scroll.

**Done:** Clicking a summary card scrolls to the corresponding section.

### Task 5: Loading state + error toast on pipeline failure (ERRES-01)
type="auto"

Wrap `start_pipeline` call in `_on_upload` (registry.py) with try/except. On exception: show error toast, restore button state, ensure app doesn't hang.

**Done:** Pipeline errors show toast, upload button re-enables, app stays responsive.

### Task 6: Graceful toast when llama-server unavailable (ERRES-02)
type="auto"

In `app/components/process.py`, catch connection errors from pipeline and show toast recommending provider switch. Also in settings.py check connection — already has error handling, but improve the message.

**Done:** When llama-server is down, user sees actionable message about switching providers.
