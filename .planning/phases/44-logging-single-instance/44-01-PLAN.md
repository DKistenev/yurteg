Create `services/log_setup.py` with a single public function `setup_logging()` that configures the root logger with two handlers:

**1. RotatingFileHandler (local):**
- Path: `~/.yurteg/logs/yurteg.log` (create parent dirs with `mkdir(parents=True, exist_ok=True)`)
- maxBytes: 5_242_880 (5 MB)
- backupCount: 3
- encoding: "utf-8"
- Format: `"%(asctime)s [%(levelname)s] %(name)s: %(message)s"`
- Level: DEBUG (capture everything locally)

**2. BetterStack Logtail handler (remote):**
- Use `logtail-python` library (pip package: `logtail-python`)
- Import: `from logtail import LogtailHandler`
- Source token from env var `BETTERSTACK_SOURCE_TOKEN` (fallback: `load_settings().get("betterstack_token", "")`)
- If token is empty string, skip adding this handler (no crash, just log a warning to file handler)
- Level: INFO (no DEBUG remotely — too noisy)
- Add `machine_id` as extra context: use `uuid.getnode()` converted to hex string (stable per machine, not PII)
- Add `app_version` from `config.APP_VERSION` as extra context

**3. Content filter for remote handler:**
Create a `logging.Filter` subclass `_ContentFilter` that rejects log records containing document content. Implementation:
- Check if `record.msg` or any of `record.args` contains suspicious patterns: strings longer than 500 chars, or records from modules `modules.text_extractor`, `modules.anonymizer` at DEBUG level
- Accept all WARNING/ERROR/CRITICAL regardless
- Accept INFO from any module
- Reject DEBUG from `modules.text_extractor` and `modules.anonymizer` (these could contain document text)
- Attach this filter ONLY to the BetterStack handler (local file keeps everything)

**Function signature:**
```python
def setup_logging() -> None:
```

Call `logging.getLogger()` (root logger), set level to DEBUG, add both handlers.

**Add to requirements.txt:**
Add line: `logtail-python>=0.3.0`

import secrets as _secrets

def _get_storage_secret() -> str:
