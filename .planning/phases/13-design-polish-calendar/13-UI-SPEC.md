## Spacing Scale

Carry forward from Phase 12 baseline — no changes in this phase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, inline padding |
| sm | 8px (`gap-2`, `p-2`) | Compact element spacing, button gaps |
| md | 16px (`gap-4`, `p-4`, `px-4`) | Default element spacing, form fields |
| lg | 24px (`gap-6`, `p-6`, `px-6`) | Section padding (existing `px-6` in registry) |
| xl | 32px (`gap-8`, `p-8`) | Layout gaps, content area padding |
| 2xl | 48px (`py-12`) | Major section breaks |
| 3xl | 64px (`py-16`) | Page-level outer spacing |

Exceptions:
- Calendar toggle button: `p-2` (8px) — icon-only button, square hit target
- FullCalendar container: `mt-4` (16px) below toggle, `p-0` on wrapper (calendar renders its own inner padding)
- Calendar event tooltip: `p-3` (12px) internal padding, `max-w-xs` (256px) width cap

**Source:** 12-UI-SPEC.md (spacing scale baseline); 13-CONTEXT.md (D-17 calendar toggle placement).

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Caption | 12px (`text-xs`) | 400 (`font-normal`) | 1.5 | Status badge labels, secondary metadata (date fields), calendar event titles at small size |
| Body | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 | All body text: table cells, form labels, descriptions, settings content, filter placeholder text |
| Heading | 20px (`text-xl`) | 600 (`font-semibold`) | 1.2 | Page-level headings, document card title, section headings in settings, onboarding headings |
| Display | 28px (`text-3xl`) | 600 (`font-semibold`) | 1.1 | `ЮрТэг` brand mark on splash screen only |

Two weights only: `font-normal` (400) for all body, captions, metadata; `font-semibold` (600) for headings, CTAs, active segment labels. No `font-medium` (500) or `font-bold` (700) anywhere.

**Font declaration (injected once in `app/main.py` via `ui.add_head_html`):**
```html

<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=IBM+Plex+Sans:wght@400;600&display=swap&subset=cyrillic" rel="stylesheet">
<style>
  * { font-family: 'IBM Plex Sans', sans-serif; }
</style>
```

**Body size note:** IBM Plex Sans at 14px and 400 weight is legible at 1.5 line-height (per 13-CONTEXT.md specifics). No bump to 15px needed — verify in implementation on actual screen before changing.

**Source:** 13-CONTEXT.md (D-06, D-07, D-08, D-09); 12-UI-SPEC.md (2-weight rule, size scale).

### 3. Calendar Toggle (new, in `app/pages/registry.py`)

Positioned: inline with segment controls row, right-aligned (`ml-auto`).

Toggle anatomy:
```
[Сегменты: Все · Истекают · Требуют внимания]        [≡]  [▦]
                                                       ↑list ↑calendar
```

Toggle button pair:
- List view button: `≡` icon (Unicode U+2630 or similar), `p-2 rounded-md`
- Calendar view button: `▦` icon (Unicode U+229E or `📅` emoji not used — use inline SVG grid icon)

### 4. FullCalendar Component (new, in `app/pages/registry.py`)

<link href='https://cdn.jsdelivr.net/npm/fullcalendar@6.1.15/index.global.min.css' rel='stylesheet' />

  eventColor: '#4f46e5',  // default indigo — overridden per event type
  buttonText: { today: 'Сегодня' },
  dayMaxEvents: 3,  // "+N more" after 3 events in a day
});
cal.render();
```

  }

```

**Tooltip on event click (D-13):** Custom HTML tooltip, not browser `alert`. Implementation: hidden `div#cal-tooltip` with `position: fixed; z-index: 1000; background: white; border: 1px solid #e2e8f0; border-radius: 8px; padding: 12px; max-width: 256px; box-shadow: 0 4px 6px -1px rgba(0,0,0,0.07)`.

Tooltip content anatomy:
```
[Тип события]                    ← text-xs text-slate-400 font-normal (Дата окончания / Платёж)
[Контрагент]                     ← text-sm text-slate-900 font-semibold
[Сумма или Срок]                 ← text-sm text-slate-600 font-normal
[Открыть →]                      ← text-sm text-indigo-600 font-semibold, cursor-pointer
```

«Открыть» click → `ui.navigate.to('/document/{contract_id}')` via JS bridge (`window.location.href`).

Tooltip dismissal: click anywhere outside closes it (document click listener).

### 5. Animations

All animations via CSS `transform` and `opacity` only — no layout property animation (per D-20).

**Staggered rows keyframe (injected in global CSS block):**
```css
@keyframes row-in {
  from { opacity: 0; transform: translateY(6px); }
  to   { opacity: 1; transform: translateY(0); }
}
.ag-row {
  animation: row-in 200ms cubic-bezier(0.25, 1, 0.5, 1) both;
}
.ag-row:nth-child(1)  { animation-delay: 0ms; }
.ag-row:nth-child(2)  { animation-delay: 80ms; }
.ag-row:nth-child(3)  { animation-delay: 160ms; }
.ag-row:nth-child(4)  { animation-delay: 240ms; }
.ag-row:nth-child(5)  { animation-delay: 320ms; }
.ag-row:nth-child(6)  { animation-delay: 400ms; }
.ag-row:nth-child(7)  { animation-delay: 480ms; }
.ag-row:nth-child(8)  { animation-delay: 560ms; }
.ag-row:nth-child(n+9) { animation-delay: 640ms; }  /* all remaining rows same delay */
```

Animation plays only on initial page load — AG Grid re-renders do not replay it (by default DOM nodes are reused, not re-created). No JS needed to trigger.

**Page fade keyframe:**
```css
@keyframes page-fade-in {
  from { opacity: 0; }
  to   { opacity: 1; }
}
.nicegui-content {
  animation: page-fade-in 200ms ease-out both;
}
```

Applied to the sub_pages content container. 200ms, ease-out (D-18).

**Hover effects (D-19):**
- Table rows: AG Grid native `rowStyle` or `getRowStyle` with `transition: background-color 150ms ease-out` — already partially covered by `_ACTIONS_CSS`; verify AG Grid supports CSS transition on hover.
- Template cards: `.classes("transition-colors duration-150 hover:bg-slate-100")` on card container
- Settings nav buttons: `.classes("transition-colors duration-150 hover:bg-slate-50")` on nav items
- Header nav tabs: `.classes("transition-colors duration-150")` on tab buttons

**Easing:** `cubic-bezier(0.25, 1, 0.5, 1)` — ease-out-quart for all entrance animations (D-21). Hover transitions use Tailwind `ease-out` (CSS `cubic-bezier(0, 0, 0.2, 1)`).

**Design intent (from 13-CONTEXT.md specifics):** Staggered reveal is subtle. If a lawyer notices it, it's too strong. Calibrate: if 80ms delay per row feels mechanical at 15+ rows, cap total stagger at 8 rows.

## Implementation Notes for Executor

1. **Font injection:** Add `ui.add_head_html(FONT_CSS)` in `app/main.py` as the first `add_head_html` call, before `_STATUS_CSS`. This ensures the font is loaded before any rendered element.

2. **FullCalendar CDN vs font:** Both are external network requests. In native mode (desktop app), these load from the internet on first render. If the app is used offline, FullCalendar will not load — acceptable for MVP, document as known limitation.

3. **Gray→slate migration:** Do NOT use sed/regex batch replace. Each replacement must be verified per-file because some `gray-900` instances are semantic text color (should become `slate-900`) and others are button background (should become `indigo-600`). The migration map in the Color section is the source of truth.

4. **`bg-gray-900` disambiguation:** Current occurrences — 4 total, all are CTA button backgrounds (`bg-gray-900 text-white font-semibold`). All 4 → `bg-indigo-600`. No `bg-gray-900` is used as text color in codebase.

5. **FullCalendar locale:** Pass `locale: 'ru'` to the Calendar constructor. jsDelivr `index.global.min.js` for v6 includes all locales. Month names, day names, and button labels will be in Russian automatically. Override `buttonText.today = 'Сегодня'` regardless.

6. **Stagger cap:** Implement nth-child delays for rows 1–8. Rows 9+ all get 640ms (flat, no further stagger). This prevents the animation from running for 2+ seconds on large registries.

7. **Hover on AG Grid rows:** AG Grid rows are DOM elements. CSS `transition` on `.ag-row:hover` works — AG Grid does not prevent it. Add `transition: background-color 150ms ease-out` to `.ag-row` in `_ACTIONS_CSS`.

8. **Calendar data injection:** In `registry.py`, when `calendar_visible` becomes True, call `run.io_bound(get_calendar_events)` to fetch payment events, then call `get_all_results()` to get `date_end` fields. Merge into one events array. Serialize to JSON. Call `ui.run_javascript(f"window.initCalendar({json_str})")`. Make `initCalendar` a global function in the injected script.
