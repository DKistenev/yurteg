## Spacing Scale

Declared values (multiples of 4px — consistent with existing NiceGUI Tailwind utility usage across Phases 7–11):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, inline padding |
| sm | 8px (`gap-2`, `p-2`) | Compact element spacing, button gaps |
| md | 16px (`gap-4`, `p-4`, `px-4`) | Default element spacing, form fields |
| lg | 24px (`gap-6`, `p-6`, `px-6`) | Section padding (matches existing `px-6` in registry) |
| xl | 32px (`gap-8`, `p-8`) | Layout gaps, content area padding |
| 2xl | 48px (`py-12`) | Splash screen vertical breathing room |
| 3xl | 64px (`py-16`) | Splash screen outer container top/bottom |

Exceptions:
- Splash screen logo area: `pb-8` (32px) below logo before heading — to create visual anchor
- Progress bar section: `py-3` (12px) vertical — compact, secondary to wizard content
- Guided tour tooltip: `p-4` (16px) internal padding, `max-w-xs` (256px) width cap

**Source:** Existing app code (`px-6 py-3 gap-4 gap-2 p-8 gap-6` patterns confirmed in registry.py, settings.py, header.py).

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 | Wizard step descriptions, progress label, tour tooltip text, empty state body, hint items, capability bullets, label items (`580/940 МБ`) |
| Heading | 20px (`text-xl`) | 600 (`font-semibold`) | 1.2 | Splash welcome heading, empty state heading, tour step title, wizard step 2 heading, app name `ЮрТэг` on splash, all CTAs |

Two weights only: `font-normal` (400) for all body, labels, descriptions, and secondary text; `font-semibold` (600) for all headings, CTAs, and the `ЮрТэг` display mark. No `font-medium` (500) or `font-bold` (700) anywhere in this phase.

The `ЮрТэг` display mark uses `text-3xl` (28px) at `font-semibold` (600) — size varies but weight does not.

**Source:** REQUIREMENTS.md (DSGN-03: чёткая типографическая иерархия); existing codebase (`text-base font-semibold`, `text-lg font-semibold`, `text-sm`, `text-xs` patterns confirmed). Checker revision 2026-03-22: consolidated from 4 weights to 2.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` / `bg-white` | Splash screen background, wizard card background, empty state area, tour tooltip background |
| Secondary (30%) | `#f9fafb` / `bg-gray-50` | Capability bullets section background, tour overlay beneath spotlight, progress section background |
| Accent (10%) | `#111827` / `bg-gray-900` | Primary CTA buttons only (consistent with existing `bg-gray-900 text-white` in segment controls) |
| Destructive | `#dc2626` / `text-red-600` | Not applicable in this phase — no destructive actions |

Accent reserved for:
- «Выбрать папку» button in empty state (only interactive CTA)
- «Далее: Telegram» / «Сохранить и начать» button in wizard steps
- «Далее» button in guided tour steps

Accent NOT used for: headings, body text, icons, progress bar fill, hint bullets, «Пропустить» links.

**Progress bar fill:** `bg-gray-700` (dark gray, not accent — progress is informational, not action).

**Tour spotlight ring:** `ring-2 ring-gray-900 ring-offset-2` around highlighted element — uses accent tone but is not a button.

**Tour overlay:** `bg-black/50` (50% black backdrop), `z-40`; highlighted element at `z-50`.

**Guided tour tooltip border:** `border border-gray-200` on white card — no shadow.

**Source:** CONTEXT.md (D-11: кнопка акцентная, подсказки серым; D-16: затемнение фона + выделение элемента); existing code (`bg-gray-900 text-white` for active segment, `bg-gray-100` for inactive).

## Component Inventory

### 1. Splash Screen (`app/components/onboarding/splash.py`)

Shown only when `first_run_completed != true` in `~/.yurteg/settings.json` (D-05).

Layout (vertical, centered, `max-w-lg mx-auto`, outer padding `py-16`):

```
[ЮрТэг]                           ← text-3xl font-semibold text-gray-900
                                     (not an image — text mark for MVP)

Добро пожаловать!                  ← text-xl font-semibold text-gray-900, mt-8

[capability bullets section]       ← bg-gray-50 rounded-lg p-4 mt-6
  · Загрузите папку → получите реестр
  · Автосортировка по папкам
  · Контроль сроков и предупреждения

[progress bar section]             ← mt-6
  Загрузка модели (580/940 МБ) — 62%   ← text-sm text-gray-500 font-normal
  [████████████░░░░░░░░]               ← ui.linear_progress, bg-gray-700 fill

[wizard section]                   ← mt-8, fade-in when rendered
  [Step 1 or Step 2 content]
```

Capability bullets: each bullet is `text-sm text-gray-600 font-normal`, preceded by `·` marker at `text-gray-400`. No icons — clean text list.

Progress bar: `ui.linear_progress(value=0.62).props("color=grey-9 track-color=grey-3").classes("w-full h-1.5 rounded-full")`.

**Source:** CONTEXT.md D-02, D-03, D-04, D-08.

### 2. Wizard Steps (inside splash, below progress bar)

**Step 1 — Приветствие** (combined with capability bullets per D-06, already rendered above).

Step 1 wizard content area shows only navigation:

```
[Пропустить]   [Далее: Telegram →]
```

`Пропустить`: `ui.button("Пропустить").props("flat no-caps").classes("text-sm text-gray-400 hover:text-gray-600")` — no background, visually secondary.

`Далее: Telegram →`: `ui.button("Далее: Telegram →").props("no-caps").classes("px-6 py-2 bg-gray-900 text-white text-sm font-semibold rounded-lg")`.

**Step 2 — Telegram**

```
Подключите Telegram-бот                ← text-xl font-semibold text-gray-900

Получайте уведомления об истекающих    ← text-sm text-gray-500 font-normal
документах прямо в мессенджер.

[Токен бота                      ]     ← ui.input, outlined dense
                                        placeholder: "110201543:AAHdqTcvCH1vGWJxfSeofSs0K"

[Пропустить]   [Сохранить и начать ✓]
```

`Сохранить и начать`: same accent style as `Далее: Telegram` — `font-semibold`. On click: saves Telegram token, sets `first_run_completed: true`, closes splash, navigates to `/`.

**Transition between steps:** `content.clear()` + re-render (NiceGUI Pattern — no animation, instant). Per FEATURES.md anti-pattern: animated transitions add latency in NiceGUI.

### 3. Empty State (registry page, `app/pages/registry.py`)

Shown in place of AG Grid when `load_table_data()` returns 0 rows AND no filters are active (D-12).

Layout (centered, `py-16 flex flex-col items-center gap-4`):

```
[folder SVG icon]                  ← 48x48px, stroke gray-300, inline SVG

Загрузите первые документы         ← text-xl font-semibold text-gray-900

Выберите папку с PDF или DOCX —    ← text-sm text-gray-500 font-normal, text-center, max-w-xs
мы извлечём метаданные и разложим
файлы автоматически.

[ Выбрать папку ]                  ← accent button, reuses pick_folder() from process.py

[hint bullets]                     ← mt-2, flex col gap-1
  · Извлечёт метаданные            ← text-sm text-gray-400 font-normal
  · Разложит по папкам
```

Folder SVG: simple outline icon — `<svg width="48" height="48" viewBox="0 0 24 24" fill="none" stroke="#d1d5db" stroke-width="1.5">` standard folder path. No emoji, no generic stock art.

«Выбрать папку» button: `bg-gray-900 text-white font-semibold`, `px-6 py-2 rounded-lg text-sm`.

After first processing: empty state removed, table shown — never returns (D-13). Flag: `first_run_completed: true` already set by wizard; table check is purely `len(rows) > 0`.

### 4. Guided Tour (`app/components/onboarding/tour.py`)

Triggered after first processing completion — when `load_table_data()` returns rows AND `tour_completed != true` (D-14).

**Overlay:** Full-screen `div` with `position: fixed; inset: 0; background: rgba(0,0,0,0.5); z-index: 40` via `ui.html`.

**Spotlight:** Target element gets `position: relative; z-index: 50; box-shadow: 0 0 0 9999px rgba(0,0,0,0.5)` injected via JS — or equivalent NiceGUI approach using `.style()`. This isolates the element visually from the dimmed overlay.

**Tooltip card:** `position: absolute` below/beside target element. Style: `bg-white rounded-lg border border-gray-200 p-4 max-w-xs shadow-sm`.

Tour step anatomy:
```
Шаг 1 / 3                          ← text-sm text-gray-400 font-normal

[step title]                        ← text-xl font-semibold text-gray-900, mb-1

[step description]                  ← text-sm text-gray-500 font-normal, leading-relaxed

[Пропустить тур]   [Далее →]       ← same button styles as wizard
```

On last step: `Далее →` becomes `Завершить`. On click: sets `tour_completed: true`, removes overlay.

**Three steps (D-15):**

| Step | Target element | Title | Description |
|------|----------------|-------|-------------|
| 1 | AG Grid table body | Реестр документов | Это ваш реестр. Кликните на строку для просмотра подробностей. |
| 2 | Segment controls + search area | Фильтры и поиск | Используйте сегменты и поиск. Истекающие документы выделены предупреждением. |
| 3 | Upload button in header | Загрузка документов | Нажмите здесь для обработки новых документов. |

Tooltip positioning:
- Step 1 (table): tooltip at `top: 120px, left: 50%, transform: translateX(-50%)` — centered above table
- Step 2 (search bar area): tooltip `below` the search row, left-aligned
- Step 3 (header upload button): tooltip `below` the button, right-aligned

**Source:** CONTEXT.md D-14, D-15, D-16, D-17, D-18.

## Interaction States

### Splash Screen

| State | Trigger | Behavior |
|-------|---------|---------|
| Loading | App start, `first_run_completed` absent/false | Splash renders full-screen, model download begins via `run.io_bound` |
| Progress updating | `on_progress` callback from `_start_llama` | `progress_bar.set_value(n/total)` + label updates via `loop.call_soon_threadsafe` |
| Model ready, wizard not done | Progress reaches 1.0 | Progress bar stays visible (filled), wizard remains for user to complete |
| Model ready + wizard done | User clicks Сохранить и начать | `save_setting('first_run_completed', True)` → `ui.navigate.to('/')` |
| Skip wizard | User clicks Пропустить at any step | `save_setting('first_run_completed', True)` → `ui.navigate.to('/')` |

### Empty State

| State | Trigger | Behavior |
|-------|---------|---------|
| Shown | `load_table_data()` returns 0 rows, no active filters | Table hidden, empty state rendered |
| Folder selected | User clicks «Выбрать папку» | Calls `pick_folder()` from `process.py`, starts pipeline inline |
| Dismissed | Pipeline completes, rows > 0 | Empty state removed, table rendered, guided tour triggered |

### Guided Tour

| State | Trigger | Behavior |
|-------|---------|---------|
| Start | First pipeline completion + `tour_completed` absent/false | Overlay renders, step 1 shown |
| Next step | User clicks «Далее» | Spotlight moves to next target element, tooltip repositions |
| Skip | User clicks «Пропустить тур» | `save_setting('tour_completed', True)`, overlay removed |
| Complete | User clicks «Завершить» on step 3 | Same as skip |
| Never shown again | `tour_completed: true` in settings | Condition check in registry `build()` after table load |

## Implementation Notes for Executor

1. **File structure:** New files — `app/components/onboarding/splash.py`, `app/components/onboarding/tour.py`. Modifications — `app/main.py` (splash gate before `ui.sub_pages`), `app/pages/registry.py` (empty state + tour trigger).

2. **Splash gate in `main.py`:** In `@ui.page('/')`, check `load_settings().get('first_run_completed')` before rendering `render_header()` + `ui.sub_pages`. If false, render splash instead.

3. **Progress wiring:** `_start_llama()` is already `async`. Add a progress callback that calls `loop.call_soon_threadsafe(progress_bar.set_value, n/total)`. `LlamaServerManager.ensure_model()` currently has no progress callback — add `on_progress: callable | None = None` parameter.

4. **Empty state trigger:** In `registry.py` `build()`, after `load_table_data()`, add: `if not rows and not any_filter_active: render_empty_state()` else render table + conditionally trigger tour.

5. **Tour overlay:** Use `ui.html('<div id="tour-overlay" ...>')` + JS to position tooltip relative to target element's `getBoundingClientRect()`. Simpler than NiceGUI native positioning for fixed-screen overlays.

6. **Settings flags:** `config.save_setting('first_run_completed', True)` and `config.save_setting('tour_completed', True)` — using existing `save_setting` from `config.py` (established in Phase 11).
