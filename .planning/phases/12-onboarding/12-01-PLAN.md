Modify `app/main.py` `root()` function to add splash gate (per RESEARCH Pattern 1):

```python
@ui.page("/")
def root() -> None:
    ui.dark_mode(value=False)
    from config import load_settings
    settings = load_settings()

    if not settings.get("first_run_completed"):
        from app.components.onboarding.splash import render_splash
        render_splash()
        return  # early return — no header, no sub_pages

    # Original code below (unchanged)
    state = get_state()
    async def _handle_upload(path):
        if hasattr(state, "_on_upload") and state._on_upload:
            await state._on_upload(path)
    render_header(state, on_upload=_handle_upload)
    ui.sub_pages({...})
```

**Key details:**
- `load_settings()` called INSIDE `root()`, not at module level (Pitfall 3)
- Early `return` prevents header + sub_pages rendering when splash is shown
- Do NOT modify `_start_llama()` — splash handles its own model download via `_run_model_download` in splash.py. The `app.on_startup(_start_llama)` still runs but checks `config.active_provider != "ollama"` — splash's download runs independently.
- Actually, to avoid double download: if splash is handling model download, `_start_llama` should be aware. Simplest approach: `_start_llama` remains unchanged — it will skip if model already downloaded (ensure_model is idempotent). If splash downloads first, `_start_llama` will see model exists and skip.
