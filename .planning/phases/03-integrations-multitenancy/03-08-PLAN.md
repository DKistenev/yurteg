**Function: `auto_bind_results(results: list[ProcessingResult], client_manager: ClientManager) -> dict`**

Logic:
1. For each result where result.success and result.metadata and result.metadata.counterparty:
   - Call client_manager.find_client_by_counterparty(result.metadata.counterparty)
   - If match found: record {filename -> matched_client_name}
   - If no match: record {filename -> None}
2. Return a summary dict:
```python
{
    "bindings": {
        "Client A": ["file1.pdf", "file2.pdf"],
        "Client B": ["file3.pdf"],
    },
    "unmatched": ["file4.pdf", "file5.pdf"],
    "total": len(results),
    "success": sum(1 for r in results if r.success),
}
```

This function does NOT move files or re-insert into different databases — it only produces the mapping. The actual data migration between client DBs (if user confirms) is a separate action triggered from the UI summary.

Place the function in controller.py (near the existing post-processing hooks like versioning/payment hooks). Import ClientManager at the top.

Also add a convenience method `move_record_to_client(record_id: int, from_db: Database, to_db: Database)` that copies a contract record from one database to another. This is needed when the user confirms auto-binding to move the record to the correct client's DB.

```python
def move_record_to_client(record_id: int, from_db: Database, to_db: Database) -> bool:
    """Copy contract record from one client DB to another."""
    record = from_db.get_contract(record_id)
    if not record:
        return False
    to_db.insert_contract(record)
    from_db.delete_contract(record_id)
    return True
```
