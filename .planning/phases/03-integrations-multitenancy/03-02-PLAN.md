**bot_server/database.py**:
ServerDatabase class with SQLite backend. Tables:
- `file_queue` (id INTEGER PK AUTOINCREMENT, chat_id INTEGER NOT NULL, file_path TEXT NOT NULL, filename TEXT NOT NULL, mime_type TEXT, enqueued_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, fetched INTEGER DEFAULT 0)
- `bindings` (chat_id INTEGER PK, bound_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP)
- `pending_bindings` (code TEXT PK, chat_id INTEGER NOT NULL, expires_at TIMESTAMP NOT NULL)
- `deadline_sync` (id INTEGER PK AUTOINCREMENT, chat_id INTEGER NOT NULL, contract_ref TEXT NOT NULL, counterparty TEXT, date_end TEXT, status TEXT, updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP)
- `notification_settings` (chat_id INTEGER PK, digest_enabled INTEGER DEFAULT 1, threshold_enabled INTEGER DEFAULT 1, warning_days INTEGER DEFAULT 30, digest_hour INTEGER DEFAULT 9)

Methods:
- `__init__(self, db_path: Path)` — creates tables if not exist. Use `check_same_thread=False` for SQLite connection (FastAPI runs in async context with multiple threads). Use `threading.Lock()` for all write operations.
- `enqueue_file(self, chat_id: int, file_path: str, filename: str, mime_type: str | None) -> int` — INSERT, returns row id
- `fetch_queue(self, chat_id: int) -> list[dict]` — SELECT WHERE chat_id=? AND fetched=0, returns list of {id, file_path, filename, mime_type, enqueued_at}
- `mark_fetched(self, file_id: int) -> None` — UPDATE SET fetched=1
- `get_binding(self, chat_id: int) -> dict | None` — SELECT from bindings
- `save_binding(self, chat_id: int) -> None` — INSERT OR REPLACE into bindings
- `save_pending_binding(self, chat_id: int, code: str, ttl_minutes: int) -> None` — INSERT with expires_at = datetime.now() + timedelta(minutes=ttl_minutes)
- `consume_pending_binding(self, code: str) -> dict | None` — SELECT WHERE code=? AND expires_at > datetime.now(), then DELETE. Return {chat_id} or None
- `save_deadlines(self, chat_id: int, alerts: list[dict]) -> None` — DELETE existing for chat_id, then INSERT new
- `get_alerts_for_user(self, chat_id: int, warning_days: int) -> list[dict]` — SELECT from deadline_sync WHERE date_end within warning_days from today
- `get_notification_settings(self, chat_id: int) -> dict` — SELECT, return defaults if not found
- `save_notification_settings(self, chat_id: int, settings: dict) -> None` — INSERT OR REPLACE

IMPORTANT: Use `check_same_thread=False` when creating the sqlite3 connection — FastAPI/uvicorn will access the DB from different threads. Use `threading.Lock()` for all write operations (same pattern as main app's modules/database.py).
Use `datetime.utcnow()` for timestamps. Use `sqlite3.Row` for row_factory.

**bot_server/bot.py**:
Create telegram bot handlers using python-telegram-bot v22.x async API.

```python
import secrets
import string
import logging
from telegram import Update
from telegram.ext import ContextTypes
from bot_server.database import ServerDatabase
from bot_server.config import QUEUE_DIR, BINDING_TTL_MINUTES, MAX_FILE_SIZE_MB

logger = logging.getLogger(__name__)

async def handle_start(update: Update, context: ContextTypes.DEFAULT_TYPE) -> None:
    """Generate 6-digit binding code and send to user."""
    chat_id = update.effective_chat.id
    db: ServerDatabase = context.bot_data["db"]
    code = ''.join(secrets.choice(string.digits) for _ in range(6))
    db.save_pending_binding(chat_id=chat_id, code=code, ttl_minutes=BINDING_TTL_MINUTES)
    await update.message.reply_text(
        f"Ваш код привязки: *{code}*\n\n"
        f"Введите его в приложении ЮрТэг -> Настройки -> Telegram.\n"
        f"Код действует {BINDING_TTL_MINUTES} минут.",
        parse_mode="Markdown"
    )

async def handle_document(update: Update, context: ContextTypes.DEFAULT_TYPE) -> None:
    """Receive document, save to queue if user is bound."""
    chat_id = update.effective_chat.id
    db: ServerDatabase = context.bot_data["db"]
    binding = db.get_binding(chat_id)
    if not binding:
        await update.message.reply_text("Сначала привяжите аккаунт — введите /start")
        return
    doc = update.message.document
    if not doc:
        await update.message.reply_text("Отправьте документ как файл (PDF или DOCX).")
        return
    # Check file size
    if doc.file_size and doc.file_size > MAX_FILE_SIZE_MB * 1024 * 1024:
        await update.message.reply_text(
            f"Файл слишком большой ({doc.file_size // (1024*1024)} МБ). "
            f"Максимум: {MAX_FILE_SIZE_MB} МБ."
        )
        return
    # Check mime type
    allowed = ("application/pdf", "application/vnd.openxmlformats-officedocument.wordprocessingml.document")
    if doc.mime_type not in allowed:
        await update.message.reply_text("Поддерживаются только PDF и DOCX файлы.")
        return
    QUEUE_DIR.mkdir(parents=True, exist_ok=True)
    dest = QUEUE_DIR / f"{chat_id}_{doc.file_unique_id}_{doc.file_name}"
    tg_file = await context.bot.get_file(doc.file_id)
    await tg_file.download_to_drive(str(dest))
    db.enqueue_file(chat_id=chat_id, file_path=str(dest), filename=doc.file_name, mime_type=doc.mime_type)
    await update.message.reply_text("Получил! Обработаю при следующем запуске приложения.")
```

**bot_server/main.py**:
FastAPI app with:
1. Lifespan context manager: init Application (python-telegram-bot), set webhook, start bot, add db to bot_data
2. POST /telegram/webhook — forward Update to bot Application
3. POST /api/bind — accept code, return chat_id via consume_pending_binding
4. GET /api/queue/{chat_id} — return list of unfetched files
5. GET /api/files/{file_id} — return file bytes (FileResponse)
6. DELETE /api/queue/{file_id} — mark file as fetched
7. POST /api/deadlines/{chat_id} — accept alerts list, save via save_deadlines
8. GET /api/health — return {"status": "ok"}

Use `from contextlib import asynccontextmanager` for lifespan.
Use `from fastapi import FastAPI, Request, HTTPException` and `from fastapi.responses import FileResponse`.
Register handlers: CommandHandler("start", handle_start), MessageHandler(filters.Document.ALL, handle_document).
Store `ServerDatabase(DB_PATH)` in `app_bot.bot_data["db"]` and also as module-level `db` for REST endpoints.
