**Implementation:**

1. After the Database object is created (look for `Database(` or `db =` in main.py), add the toast block:

```python
# --- Startup notification (INTG-04) ---
if not st.session_state.get("startup_toast_shown"):
    from services.lifecycle_service import get_attention_required
    _alerts = get_attention_required(db, config.warning_days_threshold)
    if _alerts:
        _expiring = sum(1 for a in _alerts if a.alert_type == "expiring")
        _expired = sum(1 for a in _alerts if a.alert_type == "expired")
        _parts = []
        if _expiring:
            _parts.append(f"{_expiring} истекают скоро")
        if _expired:
            _parts.append(f"{_expired} уже истекли")
        st.toast(f"Внимание: {', '.join(_parts)}", icon="warning")
    st.session_state["startup_toast_shown"] = True
```

2. Use `st.session_state` flag `startup_toast_shown` to prevent re-showing on every Streamlit rerun. Set the flag to True AFTER the toast call, regardless of whether alerts exist (so we don't re-check every rerun).

3. The `get_attention_required` function already exists in `services/lifecycle_service.py` and returns `list[DeadlineAlert]`. DeadlineAlert has `alert_type` field ("expiring" or "expired").

4. Use `icon="warning"` (Streamlit's emoji shortcode for the warning icon).

5. Do NOT create a new service or module — this is a simple UI addition using existing infrastructure.
