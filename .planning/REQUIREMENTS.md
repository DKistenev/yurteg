# Requirements: ЮрТэг v0.8.1 UI Polish

**Defined:** 2026-03-25
**Core Value:** Юрист загружает папку с документами и за 20 минут получает готовый реестр — без ручного ввода, без обучения, без «проекта внедрения»

**Visual Reference:** Все requirements имеют утверждённые HTML-мокапы в `.superpowers/brainstorm/99248-1774442361/`. Спецификация: `docs/superpowers/specs/2026-03-25-ui-polish-registry-document-design.md`

## v0.8.1 Requirements

### Реестр (REG)

- [x] **REG-01**: Колонка «Уверенность» убрана из таблицы реестра
- [x] **REG-02**: Bulk toolbar без кнопки «Экспорт в Excel», шрифт кнопок = IBM Plex Sans
- [x] **REG-03**: Pagination footer полностью на русском (без «Page Size»)
- [x] **REG-04**: Боковая панель в Linear-style: секции с мини-заголовками, тег-бейдж для типа, сумма крупно, без КАПСА
- [ ] **REG-05**: Кнопка «Календарь» открывает таймлайн событий + мини-календарь + сводку за месяц

### Карточка документа (DOC)

- [ ] **DOC-01**: Раскладка «метаданные слева (380px) + превью справа на всю высоту»
- [ ] **DOC-02**: PDF рендерится во встроенном превью (iframe), DOCX показывает извлечённый текст
- [ ] **DOC-03**: «Открыть файл» вместо «Скачать PDF» — открывает оригинал локально на компьютере
- [ ] **DOC-04**: Labels без КАПСА (кроме секционных заголовков uppercase)
- [ ] **DOC-05**: Проверка по шаблону с двумя кнопками: «Найти шаблон» (primary) + «Добавить»
- [ ] **DOC-06**: Таймлайн версий с точками (indigo/серые) и ссылками «Сравнить →»

### Диалог пространства (WS)

- [ ] **WS-01**: Карточки-пояснения (свой реестр, свои сроки, свои шаблоны) + выбор цвета из 8 + превью «так будет в меню»

### Шаблоны (TPL)

- [ ] **TPL-01**: Пустое состояние: hero + CTA + чипы-пресеты типов + 3 карточки-примера

### Настройки (SET)

- [ ] **SET-01**: Единая страница без sidebar, секции со скроллом (ИИ → Обработка → Уведомления → О программе)
- [ ] **SET-02**: Карточки модели с кольцом точности 97%, gradient bar, зелёный баннер безопасности
- [ ] **SET-03**: Thinking mode убран, резервный провайдер добавлен

### Онбординг (ONB)

- [ ] **ONB-01**: 3-шаговый wizard с прогресс-индикатором (Модель → Telegram → Готово)
- [ ] **ONB-02**: Гид-тур: tooltip с подсветкой элемента, 5 шагов, прогресс модели в карточке тура
- [ ] **ONB-03**: Кнопка «?» в header запускает тур повторно
- [ ] **ONB-04**: Временная кнопка принудительного онбординга для тестирования

### Обработка (PROC)

- [ ] **PROC-01**: Баннер прогресса: спиннер + счётчик + полоска + ETA + «Можно работать с готовыми»
- [ ] **PROC-02**: Реестр заполняется в реальном времени — новая строка с пульсацией
- [ ] **PROC-03**: Лента файлов справа (300px): ✓ готовые / ⟳ текущий / ! ошибки / очередь

## Deferred (v0.9+)

- Сборка DMG для macOS
- Сборка EXE для Windows
- Автообновление
- OCR для сканированных PDF
- Поиск шаблона по векторам (кнопка «Найти» готова, бэкенд позже)
