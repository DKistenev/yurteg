# Cross-AI Roadmap Review — v0.7 Визуальный продукт

## Architecture Review

### Summary
The v0.7 roadmap is well-researched and architecturally sound at the strategy level — the CSS token hierarchy, load order, and pitfall catalogue are thorough and sourced from real NiceGUI issues. The 3-phase structure works for velocity, but Phase 15 is materially oversized: it packs 3 distinct surfaces (splash, registry, document card) plus AG Grid theming into a single gate, which creates a wide blast radius if the AG Grid work stalls. The known gaps identified in the audit are genuine blockers, not cosmetic — particularly the missing body background requirement (DSGN-05) and the NiceGUI padding reset omission, both of which will cause silent visual failures if not added to Phase 14 before any downstream work starts.

### Strengths
- Token-first dependency ordering is correct — every downstream visual change inherits from the token layer
- AG Grid theming isolated to its own phase (15) — highest-risk CSS work
- Functional class freeze rule explicitly documented (actions-cell, status-*)
- backdrop-filter: blur() explicitly excluded with technical rationale
- --yt-* prefix necessity for FullCalendar collision avoidance
- Phase 16 deferred correctly — animations after structure is stable
- ui.colors() anti-pattern documented (per-page vs once in root)

### Concerns
- **[HIGH]** DSGN-05 (body/content background) is missing — Phase 14 blocker
- **[HIGH]** Phase 15 is too wide — 11 req, 3 surfaces, AG Grid blocking splash/card
- **[HIGH]** --nicegui-default-padding: 0 reset not in any requirement
- **[MEDIUM]** TMPL-01 describes anti-pattern per design principles
- **[MEDIUM]** REGI-06 (filter bar) missing — visual seam with stats bar
- **[MEDIUM]** Animation double-fire on filter switch (Pitfall 7) not guarded
- **[MEDIUM]** FullCalendar --fc-* variable enumeration not completed
- **[LOW]** Phase 15 criterion 4 (card breadcrumbs) depends on registry state
- **[LOW]** ANIM-03/04 in Phase 16 have no performance budget
- **[LOW]** All plans TBD — CSS load order is order-sensitive

### Suggestions
- Promote DSGN-05 and --nicegui-default-padding reset to Phase 14
- Split Phase 15 into 15a (Splash) and 15b (Registry + Card)
- Add REGI-06 (filter bar) to Phase 15
- Add Phase 15 criterion: "Animation does not replay on filter switch"
- Sharpen TMPL-01: color-coded accent, type icon, not just shadow+rounded
- Pre-Phase 15 DevTools checkpoint for AG Grid theme class name
- Add --fc-* variable enumeration to Phase 14 smoke-test
- Add "no jank on macOS pywebview" as Phase 16 criterion

### Risk Assessment
**MEDIUM** — Research thorough, architecture correct, but Phase 15 too wide and two foundational requirements missing from Phase 14. If both gaps addressed, drops to LOW.

## Design Director Review

### Summary
The roadmap is structurally sound — three-phase sequencing follows correct dependency order, "dark chrome + light content" is the right call, anti-features are explicitly rejected. However, requirements are not specific enough to prevent generic output. Several requirements are written at "add a shadow" level rather than specifying contrast ratios, size mappings, differentiation logic. Phase 15 is too wide. Two anti-patterns are baked in: TMPL-01 (rounded-xl + shadow-sm = generic card) and CARD-03 (identical metadata cards).

### Strengths
- Dependency ordering correct — tokens → components → polish
- "Dark chrome + light content" explicitly documented with rationale
- Anti-features named and rejected with reasons (glassmorphism, sidebar, parallax)
- backdrop-filter exclusion with specific technical reason
- --yt-* prefix decision is solid defensive engineering
- FEATURES.md reference table (RunPod/Linear/Notion/Vercel) gives concrete anchors

### Concerns
- **[HIGH]** TMPL-01 literally encodes an anti-pattern — generic card format
- **[HIGH]** Phase 15 has 11 req across 3 unrelated surfaces — waterfall risk
- **[HIGH]** DSGN-05 missing — most visible problem (white voids) has no requirement
- **[HIGH]** No typography size spread enforcement — DSGN-03 lacks role mapping
- **[MEDIUM]** CARD-03 identical-card-grid risk — 3 content types, same shadow
- **[MEDIUM]** --nicegui-default-padding reset missing from all requirements
- **[MEDIUM]** REGI-06 (filter bar) missing — visual seam
- **[MEDIUM]** ANIM-02 dependency on Phase 15 HTML structure is implicit
- **[LOW]** FullCalendar criterion dilutes Phase 14 design intent
- **[LOW]** SPLS-03 (stagger animation) should be stretch, not core

### Suggestions
- Split Phase 15 into 15A (Splash only) and 15B (Registry + Card)
- Add DSGN-05: content area background ≠ white, specify token value
- Rewrite TMPL-01: color-coded left-border, type icon, visual differentiation
- Add font size role mapping to DSGN-03 (hero=4xl, title=2xl, section=xs uppercase, body=sm)
- Add REGI-06 to Phase 16 or document why out of scope
- Mark SPLS-03 as stretch goal, not core
- Add visual seam cross-cut check: coherence when navigating header→splash→registry→card→settings
- Add DSGN-05 and padding reset to Phase 14

### Risk Assessment
**MEDIUM-HIGH** — Foundational work well-structured. Risk concentrated in Phase 15 overload and DSGN-05 missing. TMPL-01 will produce generic output as written. Fixable gaps, not fundamental errors.

## Consensus Summary

### Agreed Strengths (both reviewers)
- Token-first dependency ordering is correct
- "Dark chrome + light content" framing is the right design decision
- Anti-features explicitly rejected with rationale
- backdrop-filter exclusion with technical justification
- --yt-* prefix for FullCalendar collision avoidance
- Research quality is unusually high for an internal plan

### Agreed Concerns (both reviewers — highest priority)
1. **[HIGH] DSGN-05 missing** — No requirement for body/content background ≠ white. Both reviewers flag this as the most visible problem with no coverage.
2. **[HIGH] Phase 15 overloaded** — 11 requirements, 3 unrelated surfaces. Both recommend splitting: splash first (pattern validation), then registry + card.
3. **[HIGH] TMPL-01 anti-pattern** — Both flag "shadow-sm, rounded-xl" as generic. Need differentiation: color-coded accents, type icons.
4. **[HIGH] --nicegui-default-padding reset missing** — Both flag this as a foundational omission for Phase 14.
5. **[MEDIUM] REGI-06 filter bar missing** — Both note the visual seam risk.
6. **[MEDIUM] Typography role mapping missing** — Design director specifically flags DSGN-03 lacks size-to-role spec.

### Divergent Views
- **Phase 15 split granularity:** Architecture reviewer suggests 15a/15b (splash vs registry+card). Design director suggests the same split but additionally flags SPLS-03 as a stretch goal.
- **FullCalendar smoke-test placement:** Architecture reviewer sees it as valid success criterion. Design director sees it as regression test that dilutes design intent. Both agree it should happen, just disagree on framing.
- **Overall risk:** Architecture = MEDIUM. Design Director = MEDIUM-HIGH. Delta driven by design specificity concerns (TMPL-01, typography roles) that architecture reviewer noted but weighted lower.
