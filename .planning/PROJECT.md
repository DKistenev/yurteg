# ЮрТэг — Веха 1: Архитектура и функционал

## What This Is

ЮрТэг — десктопное приложение на Python (Streamlit) для автоматической обработки архивов юридических документов. Пользователь загружает папку с PDF/DOCX → приложение извлекает текст → анонимизирует ПД → отправляет в LLM для извлечения метаданных → раскладывает файлы по папкам → генерирует Excel-реестр. MVP собран, работает локально. Команда: 3 юриста, нет разработчика — всё через Claude Code.

## Core Value

Юрист загружает папку с документами и за 20 минут получает готовый реестр с метаданными — без ручного ввода, без обучения, без «проекта внедрения».

## Context

### CustDev-результаты (19.03.2026)

9 интервью (3 реальных + 6 синтетических). Ключевые находки:

**Подтверждённые боли:**
- Поиск чужих/старых документов — от 10 мин до 1.5 часов (9 из 9)
- Ручное ведение реестра — бессмысленная рутина (7 из 9)
- Пропущенные сроки и потеря контроля (4 из 9, с историями потерь — но цифры из синтетических интервью, требуют валидации)
- Хаос нейминга файлов (6 из 9)
- Документы раскиданы по 3–5 местам (7 из 9)

**Главные барьеры:**
1. Безопасность — снимается локальностью (~80%) + open source (+10–15%)
2. «Ещё одна система» — снимается zero-onboarding подходом
3. Недоверие к AI — снимается подсветкой неуверенности + быстрой перепроверкой
4. Ценовой порог — бесплатный период обязателен

**Сегменты по LTV (убывание):**
- Руководители юротделов — высший LTV, сложный вход (PTSD после СЭД)
- Крупный инхаус — процедурный барьер, нужен on-premise
- МСБ юристы — не ЛПР, нужен аргумент для директора
- Юристы-аутсорсеры — прагматичны, канал дистрибуции
- Tech-savvy юристы — early adopters, хотят API
- Микробизнес/ИП — не наш клиент при текущем позиционировании

**Важно:** финансовые потери из интервью (670к+) — синтетические данные. Паттерн пропуска сроков подтверждён, но конкретные суммы требуют проверки в реальных интервью.

### Технический контекст

- MVP работает на Streamlit, пайплайн модульный (scanner → extractor → anonymizer → ai_extractor → validator → database → organizer → reporter)
- AI: ZAI GLM-4.7 через openai SDK (выбрана по цене), OpenRouter как fallback
- Локальная LLM: дообученная QWEN 7B, планируется дистилляция до 1.5B (отдельная веха)
- БД: SQLite
- Карта кодовой базы: `.planning/codebase/`

## Constraints

- **Команда**: 3 юриста, нет разработчика — всё через Claude Code
- **Tech stack**: Python, Streamlit, SQLite — менять стек нецелесообразно
- **AI SDK**: openai Python SDK (не anthropic) — для совместимости с GLM и OpenRouter
- **Безопасность**: локальная обработка по умолчанию, анонимизация как дополнительный слой
- **Хакатон**: участвуем, но сроки не давят — «всё успеваем»
