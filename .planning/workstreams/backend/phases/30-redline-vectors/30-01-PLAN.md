**Алгоритм (word-level):**

1. Токенизация: `re.findall(r'\S+|\s+', text)` — сохраняет пробелы как отдельные токены, чтобы не терять пробелы между словами в track changes.

2. difflib.SequenceMatcher(None, tokens_old, tokens_new).get_opcodes() — на токенах, не предложениях.

3. Генерация XML:
   - `equal` → обычный `para.add_run(token)`
   - `delete` → `w:del` + `w:r` + `w:delText xml:space="preserve"`, текст = конкатенация удалённых токенов. rPr = `OxmlElement('w:rPr')` (пустой, т.к. plain text; если в будущем нужно копировать — вставить перед `w:delText`).
   - `insert` → `w:ins` + `w:r` + `w:t xml:space="preserve"`, текст = конкатенация вставленных токенов. rPr аналогично.
   - `replace` → сначала delete-блок, затем insert-блок в одном параграфе.

4. Разбивка по абзацам: перед вызовом SequenceMatcher разбить оба текста на параграфы через `text.split('\n')`. Каждый параграф обрабатывать отдельно → `doc.add_paragraph()` между ними.

**Критические детали OOXML (per research Pitfall 4):**
- Удалённый контент ОБЯЗАТЕЛЬНО использует `w:delText` (не `w:t`), иначе Word показывает диалог восстановления.
- `w:ins` и `w:del` — прямые дети параграфа `w:p`, не обёртки вокруг `w:r`.
- `w:id` должен быть уникальным int (использовать счётчик `itertools.count(1)`).
- `w:author` = "ЮрТэг", `w:date` = "2026-01-01T00:00:00Z".
