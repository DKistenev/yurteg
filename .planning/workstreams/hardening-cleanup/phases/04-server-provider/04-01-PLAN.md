- `HEALTH_ENDPOINT = f"http://localhost:{DEFAULT_PORT}/health"`

**Класс LlamaServerManager:**

```python
class LlamaServerManager:
    def __init__(self, port: int = DEFAULT_PORT):
        self._port = port
        self._process: subprocess.Popen | None = None
        self._yurteg_dir = YURTEG_DIR
        self._yurteg_dir.mkdir(parents=True, exist_ok=True)
```

**Методы:**

1. `ensure_model(on_progress: Callable[[float, str], None] | None = None) -> Path`:
   - Проверяет `~/.yurteg/yurteg-v3-Q4_K_M.gguf` существует
   - Если нет — скачивает через `huggingface_hub.hf_hub_download(repo_id=MODEL_REPO, filename=MODEL_FILENAME, cache_dir=str(YURTEG_DIR), local_dir=str(YURTEG_DIR))`
   - Callback `on_progress(fraction, message)` для UI прогресс-бара (per D-07)
   - Возвращает Path к файлу модели

2. `ensure_server_binary(on_progress: Callable[[float, str], None] | None = None) -> Path`:
   - Проверяет `~/.yurteg/llama-server` (или `.exe` на Windows) существует
   - Если нет — скачивает с GitHub releases llama.cpp (per D-01, D-08)
   - Определяет платформу через `platform.system()` + `platform.machine()`
   - macOS arm64: `llama-bXXXX-bin-macos-arm64.zip` → извлечь `build/bin/llama-server`
   - macOS x86: `llama-bXXXX-bin-macos-x64.zip`
   - Linux: `llama-bXXXX-bin-ubuntu-x64.zip`
   - Windows: `llama-bXXXX-bin-win-cuda-cu12.2.0-x64.zip` (or avx2 for non-CUDA)
   - Распаковать нужный бинарник, сделать chmod +x
   - Возвращает Path к бинарнику

3. `start(grammar_path: Path | None = None) -> None`:
   - Запускает llama-server как subprocess (per D-04)
   - Команда: `[binary_path, "-m", model_path, "--port", str(port), "-c", "4096", "-n", "512", "--temp", "0.05", "--min-p", "0.05", "--top-p", "1.0", "--repeat-penalty", "1.1"]`
   - Если grammar_path передан: добавить `["--grammar-file", str(grammar_path)]`
   - stdout/stderr перенаправить в subprocess.DEVNULL (не засорять логи)
   - Ожидание готовности: poll `HEALTH_ENDPOINT` каждые 0.5 сек, таймаут 30 сек
   - При ошибке старта — raise RuntimeError с сообщением
   - Зарегистрировать `atexit.register(self.stop)` (per D-05)

4. `stop() -> None`:
   - Если `self._process` жив — `self._process.terminate()`, `self._process.wait(timeout=5)`
   - При таймауте — `self._process.kill()`
   - `self._process = None`

5. `is_running() -> bool`:
   - Проверяет `self._process` и `self._process.poll() is None`
   - Дополнительно GET на HEALTH_ENDPOINT (requests/urllib)

6. `@property base_url(self) -> str`:
   - Возвращает `f"http://localhost:{self._port}/v1"` (OpenAI-совместимый endpoint)

**Обработка конфликтов портов:** Если порт занят (OSError при старте), попробовать порт + 1, до 3 попыток.

**Fallback (per D-06):** Не реализовывать в этом модуле — fallback на облако обрабатывается в providers/__init__.py (get_provider вернёт fallback если ollama не стартовал). В `start()` при ошибке логировать warning и не raise (позволить fallback сработать).

**Зависимости:** Добавить `huggingface_hub` в requirements.txt (для скачивания модели). Для HTTP health check использовать `urllib.request` из стандартной библиотеки (не добавлять requests).

**1. Создать `data/contract.gbnf` (per D-09):**

GBNF грамматика для llama-server, описывающая валидный JSON объект ContractMetadata:
```
root   ::= "{" ws members ws "}"
members ::= pair ("," ws pair)*
pair   ::= ws string ws ":" ws value ws
...
```

Грамматика должна разрешать:
- Все поля ContractMetadata как строки или null
- `payment_frequency` строго enum: `"monthly"` | `"quarterly"` | `"yearly"` | `"once"` | null
- `payment_direction` строго enum: `"income"` | `"expense"` | null
- `confidence` — число от 0.0 до 1.0
- `is_template` — boolean
- `parties`, `special_conditions` — массивы строк
- `payment_amount` — число или null
- Строковые значения — UTF-8 (кириллица + латиница + цифры + пробелы + пунктуация)

**2. Создать `modules/postprocessor.py` (per D-10, D-11):**

```python
"""Post-processing ответов локальной модели.

Каждое поле ContractMetadata имеет свой профиль допустимых символов.
Санитайзер применяется после получения ответа от llama-server.
"""

FIELD_PROFILES: dict[str, str] = {
    # "cyrillic_only" — только кириллица + пробелы + пунктуация
    "contract_type": "cyrillic_only",
    "special_conditions": "cyrillic_only",
    # "cyrillic_latin" — кириллица + латиница + цифры + пробелы + пунктуация
    "counterparty": "cyrillic_latin",
    "parties": "cyrillic_latin",
    "subject": "cyrillic_latin",
    "amount": "cyrillic_latin",
    "payment_terms": "cyrillic_latin",
    # "enum" — строго из допустимого набора
    "payment_frequency": "enum",
    "payment_direction": "enum",
    # "date" — YYYY-MM-DD или null
    "date_signed": "date",
    "date_start": "date",
    "date_end": "date",
    # "number" — float или null
    "confidence": "number",
    "payment_amount": "number",
    # "boolean"
    "is_template": "boolean",
}

ENUM_VALUES = {
    "payment_frequency": {"monthly", "quarterly", "yearly", "once"},
    "payment_direction": {"income", "expense"},
}
```

Функция `sanitize_metadata(raw: dict) -> dict`:
- Для каждого поля применяет профиль:
  - `cyrillic_only`: regex `re.sub(r'[^\u0400-\u04FFёЁ\s\d.,;:!?()«»""—–\-/№]', '', value)` — удаляет латиницу
  - `cyrillic_latin`: regex `re.sub(r'[^\u0400-\u04FFёЁa-zA-Z\s\d.,;:!?()«»""—–\-/№%₽€$]', '', value)` — оставляет оба алфавита
  - `enum`: если значение не в ENUM_VALUES[field] → None
  - `date`: regex проверка `r'^\d{4}-\d{2}-\d{2}$'`, иначе None
  - `number`: попытка float(), иначе None/0.0
  - `boolean`: bool()
- Для list-полей (`parties`, `special_conditions`) — применить профиль к каждому элементу
- Строки `"None"`, `"null"`, `""` → None (per PROC-01 подготовка)
- Возвращает очищенный dict

Функция `get_grammar_path() -> Path`:
- Возвращает `Path(__file__).parent.parent / "data" / "contract.gbnf"`
