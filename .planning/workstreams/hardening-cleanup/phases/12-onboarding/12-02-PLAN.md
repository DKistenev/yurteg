**Implementation via ui.html + JS (per RESEARCH Pattern 5):**

`render_tour()` injects a single `ui.html` block containing:
1. Overlay div: `position: fixed; inset: 0; background: rgba(0,0,0,0.5); z-index: 40`
2. Tooltip card div: `position: fixed; z-index: 50; background: white; border: 1px solid #e5e7eb; border-radius: 8px; padding: 16px; max-width: 256px`
3. JS functions:
   - `startTour()` — shows overlay, highlights first step
   - `showStep(index)` — positions tooltip relative to target element via `getBoundingClientRect()`, applies spotlight (`box-shadow: 0 0 0 9999px rgba(0,0,0,0.5); position: relative; z-index: 50`) on target
   - `nextStep()` — increments step, clears previous spotlight, calls showStep
   - `endTour()` — removes overlay, clears spotlights, calls NiceGUI backend to save flag

**Tooltip card HTML structure** (per UI-SPEC):
- Step counter: `<div style="font-size: 14px; color: #9ca3af;">Шаг {N} / 3</div>`
- Title: `<div style="font-size: 20px; font-weight: 600; color: #111827; margin-bottom: 4px;">{title}</div>`
- Body: `<div style="font-size: 14px; color: #6b7280; line-height: 1.625;">{body}</div>`
- Buttons row:
  - Left: «Пропустить тур» — `font-size: 14px; color: #9ca3af; cursor: pointer; background: none; border: none`
  - Right: «Далее →» (steps 1-2) or «Завершить тур» (step 3) — same accent style: `padding: 8px 24px; background: #111827; color: white; font-size: 14px; font-weight: 600; border-radius: 8px; border: none; cursor: pointer`

**NiceGUI-JS bridge for saving flag:**
Use `ui.run_javascript` is not needed inside tour.py. Instead, the tour HTML should call a JS function that triggers an endpoint. Simpler approach: `render_tour()` returns a reference, and the caller (registry.py) handles the save. OR: use NiceGUI's `ui.on('tour_complete', ...)` pattern.

Simplest: render_tour takes an `on_complete` callback. In the JS, when tour ends, it calls `emitEvent('tour_complete')`. In Python: `ui.on('tour_complete', lambda: save_setting('tour_completed', True))`.

Actually, simplest NiceGUI pattern: the endTour JS calls `fetch('/api/tour-complete')` or use `nicegui.globals.client`. Better: just make `render_tour` accept an `on_complete` async function. Use `ui.button` with `visibility: hidden` that JS clicks programmatically when tour ends. Or use `app.storage.general`.

Cleanest approach: render_tour creates hidden NiceGUI buttons for "next" and "skip" that JS clicks:
```python
def render_tour(on_complete: Callable) -> None:
    done_btn = ui.button("", on_click=on_complete).classes("hidden")
    # JS endTour() calls: document.querySelector('#tour-done-btn').click()
```
Give the hidden button an HTML id via `.props('id=tour-done-btn')`.
