# UI Polish: Registry + Document Card — Design Spec

**Date:** 2026-03-25
**Status:** Approved
**Source:** Visual brainstorming session with mockups

## Goal

Исправить визуальные проблемы реестра и карточки документа. Убрать пустоты, починить стиль, добавить предпросмотр документа.

## Design Decisions

### Реестр

**Bulk actions toolbar:**
- Убрать кнопку «Экспорт в Excel» — не нужна
- Кнопки: «Изменить статус», «Удалить», «Снять выбор»
- Шрифт кнопок = IBM Plex Sans (как всё приложение), не системный
- Стиль: border border-slate-200, rounded-md, text-slate-600, font-size 12px

**Таблица:**
- Убрать колонку «Уверенность» из таблицы — внутренняя метрика, не для пользователя
- Колонки: ☐, Тип, Контрагент, Предмет, Статус, Сумма

**Pagination footer:**
- Русифицировать: «Page Size:» → убрать, «1 до 10 из 10» оставить, «Стр. 1 из 1» оставить
- Стиль: мелкий серый текст, аккуратный

**Боковая панель предпросмотра (Linear-style):**
- Заголовок: контрагент крупным + кнопка ✕ закрыть
- Тег-бейдж: тип документа как indigo метка
- Секции с мини-заголовками uppercase: ДОКУМЕНТ, СРОКИ, ФИНАНСЫ
- Статус badge в секции «Документ»
- Даты в 2-колоночном grid
- Сумма крупным шрифтом (20px, font-weight 700) в отдельной секции
- Кнопка «Открыть карточку →» внизу
- Labels БЕЗ КАПСА (обычный регистр, серый цвет)

**Календарь:**
- Починить кнопку «Календарь» — сейчас показывает пустой экран

### Карточка документа

**Раскладка: две колонки на всю высоту экрана**
- Левая панель (380px, фиксированная, скроллируется): все метаданные и действия
- Правая панель (flex: 1): предпросмотр документа на всю высоту

**Топ-бар (отдельная полоса сверху):**
- Breadcrumbs: «Реестр → Тип документа»
- Навигация ◂ ▸ (пред/след документ)
- Справа: кнопка «📂 Открыть файл» + «↻ Переобработать»
- «Открыть файл» = открыть оригинал локально на компьютере (не скачивать)

**Левая панель — порядок секций:**
1. **Документ** — тип, контрагент (bold), предмет
2. **Сроки и финансы** — начало, окончание (2 колонки), подписание, сумма (16px bold)
3. **Статус** — badge + ссылка «Изменить»
4. --- разделитель ---
5. **Проверка по шаблону** — две кнопки: «🔍 Найти» (primary indigo) + «+ Добавить» (outlined)
6. --- разделитель ---
7. **История версий** — таймлайн с точками (indigo = текущая, серые = старые), ссылки «Сравнить →»
8. --- разделитель ---
9. **Пометки юриста** — textarea

**Правая панель — предпросмотр документа:**
- Тёмный фон (#1e293b)
- Toolbar сверху: имя файла + «стр. X из Y» + кнопки ◂ ▸ ⊕ ⊖
- PDF: встроенный рендер через iframe или pdf.js
- DOCX: отображать извлечённый текст (уже есть в базе после обработки)
- Белая «страница» по центру с тенью

**Стиль labels:**
- Секционные заголовки: uppercase, 10px, spacing 0.05em, цвет #94a3b8
- Field labels: обычный регистр (не КАПС), 10-11px, цвет #94a3b8
- Field values: 13px, цвет #1e293b

### Диалог «Новое пространство»

**Полный редизайн — A: карточки + цвет + превью:**
- Убрать жирную indigo-шапку → белый фон, ✕ закрыть
- Заголовок «Новое пространство» + подзаголовок
- 3 карточки-пояснения:
  - 📁 Свой реестр — «Документы не смешиваются между клиентами»
  - 🔔 Свои сроки — «Напоминания отдельно по каждому клиенту»
  - 📋 Свои шаблоны — «Набор шаблонов под тип клиента»
- Поле «Название» с placeholder «Например: ООО «Ромашка»»
- Выбор цвета — 8 кружков (indigo, cyan, green, amber, red, violet, pink, slate)
- Превью «Так будет в меню» — точка цвета + название
- Кнопки: «Отмена» (серая) + «Создать» (indigo filled)
- Цвет сохранять в client config и использовать в dropdown header

**Files:** `app/components/header.py` (диалог создания), `app/state.py` (если нужно поле цвета)

### Календарь (вариант B: таймлайн + мини-календарь)

**Полный редизайн вида «Календарь»:**
- Слева: лента событий, группированная по времени (Сегодня → Эта неделя → Следующий месяц)
- Каждое событие — карточка с цветной полоской слева (красная = окончание, синяя = платёж, жёлтая = скоро истекает)
- На карточке: тип события, контрагент, предмет, сумма крупно справа, дата
- Справа: мини-календарь с цветными точками на датах + сводка за месяц (сколько истекает, платежей, сумма платежей)
- Клик по карточке → переход в карточку документа

**Files:** `app/pages/registry.py` (calendar view section)

### Шаблоны — пустое состояние (вариант B: hero + пресеты + примеры)

**Когда шаблонов нет:**
- Hero: иконка 📋 + «Добавьте первый шаблон» + описание что это и зачем + CTA «Загрузить шаблон»
- Чипы-пресеты: Поставка, Аренда, Трудовой, Услуги, Займ, Подряд, NDA — клик = добавить шаблон этого типа
- 3 карточки-примера внизу: 2 заполненные (как будут выглядеть) + 1 placeholder «Ваш шаблон будет здесь»

**Files:** `app/pages/templates.py` (empty state section)

### Настройки (единая страница + красивые карточки модели)

**Полный редизайн — единая страница со скроллом (без sidebar):**
- Заголовок «Настройки» + подзаголовок
- Секция «ИИ-модель»:
  - 2 карточки: модель (QWEN 1.5B, gradient bar, статус) + точность (кольцо 97%, скорость, офлайн)
  - Зелёный баннер «🔒 Данные обрабатываются локально»
  - Карточка настроек: провайдер (dropdown), сервер (badge + проверить), резервный провайдер
  - Thinking mode УБРАН (всегда отключен)
- Секция «Обработка»: анонимизация (toggle), порог уверенности, параллельные потоки
- Секция «Уведомления»: Telegram (badge + привязать), порог напоминания
- Секция «О программе»: версия, обработано документов, размер базы

**Files:** `app/pages/settings.py` (полная переработка)

### Онбординг (вариант A: пошаговый wizard)

**Доработка существующего онбординга:**
- 3-шаговый прогресс сверху: Модель (✓) → Telegram (●) → Готово (○)
- Карточка текущего шага по центру на тёмном hero
- Прогресс модели внизу с gradient bar
- Шаг 3: после Telegram → запускается гид-тур по приложению

**Files:** `app/components/onboarding/splash.py`

### Гид-тур (вариант A: tooltip с подсветкой)

**Интерактивный тур внутри приложения:**
- Затемнение всего экрана кроме подсвеченного элемента (box-shadow trick)
- Tooltip-карточка рядом: шаг X из 5, заголовок, описание, точки, «Пропустить / Далее →»
- Прогресс загрузки модели прямо в карточке тура — юрист видит что скачивание идёт
- 5 шагов: загрузка документов → навигация → фильтры/поиск → календарь → рабочие пространства
- Запускается автоматически после онбординга ИЛИ по кнопке «?» в header
- Временная кнопка «🔄 Онбординг» в header для тестирования (убрать перед релизом)
- Селекторы через data-tour атрибуты (уже расставлены в v0.8)

**Files:** `app/components/onboarding/tour.py`, `app/components/header.py`

### Экран обработки (живой реестр + лента файлов)

**Во время обработки документов:**
- Баннер прогресса сверху: спиннер + «7 из 24» + полоска + ETA + «Можно работать с готовыми»
- Слева: реестр заполняется в реальном времени — новая строка с пульсацией (animation: rowIn), текущий файл внизу таблицы со спиннером, статистика растёт
- Справа: лента файлов (300px):
  - ✓ готовые (с извлечёнными метаданными и временем обработки)
  - ⟳ текущий (спиннер + название файла)
  - ! ошибки (красным, с описанием)
  - Очередь внизу (список файлов ожидающих обработки)
- Юрист может кликать по готовым строкам, открывать карточки, работать параллельно

**Files:** `app/pages/registry.py` (processing section), `app/components/registry_table.py`

## Acceptance Criteria

1. Реестр: колонка «Уверенность» отсутствует в таблице
2. Реестр: bulk toolbar без кнопки Excel, кнопки с правильным шрифтом
3. Реестр: pagination footer полностью на русском
4. Реестр: боковая панель в Linear-style с секциями
5. Реестр: кнопка «Календарь» открывает таймлайн + мини-календарь
6. Карточка: метаданные слева, превью справа на всю высоту
7. Карточка: PDF рендерится во встроенном preview
8. Карточка: DOCX показывает извлечённый текст
9. Карточка: «Открыть файл» открывает оригинал локально
10. Карточка: labels без КАПСА (кроме секционных заголовков)
11. Карточка: шаблон с двумя кнопками «Найти» + «Добавить»
12. Карточка: таймлайн версий с точками и ссылками «Сравнить»
13. Диалог: карточки-пояснения + выбор цвета + превью
14. Календарь: таймлайн событий слева + мини-календарь справа + сводка
15. Шаблоны: hero + чипы-пресеты + примеры карточек при пустом состоянии
16. Настройки: единая страница, карточки модели с кольцом точности, без Thinking
17. Настройки: секция «О программе» с версией и счётчиком документов
18. Онбординг: 3-шаговый wizard с прогрессом
19. Гид-тур: tooltip с подсветкой элемента, 5 шагов, прогресс модели в карточке
20. Гид-тур: кнопка «?» в header запускает тур повторно
21. Обработка: реестр заполняется в реальном времени с пульсацией
22. Обработка: лента файлов справа (✓/⟳/!/очередь)
23. Обработка: баннер прогресса с ETA и подсказкой «можно работать»
24. Тест: временная кнопка принудительного онбординга в header

## Constraints

- NiceGUI 3.x: CSS в @layer components / @layer overrides
- AG Grid: изменения через columnDefs
- Функциональные class names (status-active и т.д.) — не переименовывать
- Не менять backend логику — только UI
