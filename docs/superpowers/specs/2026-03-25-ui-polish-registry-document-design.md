# UI Polish: Registry + Document Card — Design Spec

**Date:** 2026-03-25
**Status:** Approved
**Source:** Visual brainstorming session with mockups

## Goal

Исправить визуальные проблемы реестра и карточки документа. Убрать пустоты, починить стиль, добавить предпросмотр документа.

## Design Decisions

### Реестр

**Bulk actions toolbar:**
- Убрать кнопку «Экспорт в Excel» — не нужна
- Кнопки: «Изменить статус», «Удалить», «Снять выбор»
- Шрифт кнопок = IBM Plex Sans (как всё приложение), не системный
- Стиль: border border-slate-200, rounded-md, text-slate-600, font-size 12px

**Таблица:**
- Убрать колонку «Уверенность» из таблицы — внутренняя метрика, не для пользователя
- Колонки: ☐, Тип, Контрагент, Предмет, Статус, Сумма

**Pagination footer:**
- Русифицировать: «Page Size:» → убрать, «1 до 10 из 10» оставить, «Стр. 1 из 1» оставить
- Стиль: мелкий серый текст, аккуратный

**Боковая панель предпросмотра (Linear-style):**
- Заголовок: контрагент крупным + кнопка ✕ закрыть
- Тег-бейдж: тип документа как indigo метка
- Секции с мини-заголовками uppercase: ДОКУМЕНТ, СРОКИ, ФИНАНСЫ
- Статус badge в секции «Документ»
- Даты в 2-колоночном grid
- Сумма крупным шрифтом (20px, font-weight 700) в отдельной секции
- Кнопка «Открыть карточку →» внизу
- Labels БЕЗ КАПСА (обычный регистр, серый цвет)

**Календарь:**
- Починить кнопку «Календарь» — сейчас показывает пустой экран

### Карточка документа

**Раскладка: две колонки на всю высоту экрана**
- Левая панель (380px, фиксированная, скроллируется): все метаданные и действия
- Правая панель (flex: 1): предпросмотр документа на всю высоту

**Топ-бар (отдельная полоса сверху):**
- Breadcrumbs: «Реестр → Тип документа»
- Навигация ◂ ▸ (пред/след документ)
- Справа: кнопка «📂 Открыть файл» + «↻ Переобработать»
- «Открыть файл» = открыть оригинал локально на компьютере (не скачивать)

**Левая панель — порядок секций:**
1. **Документ** — тип, контрагент (bold), предмет
2. **Сроки и финансы** — начало, окончание (2 колонки), подписание, сумма (16px bold)
3. **Статус** — badge + ссылка «Изменить»
4. --- разделитель ---
5. **Проверка по шаблону** — две кнопки: «🔍 Найти» (primary indigo) + «+ Добавить» (outlined)
6. --- разделитель ---
7. **История версий** — таймлайн с точками (indigo = текущая, серые = старые), ссылки «Сравнить →»
8. --- разделитель ---
9. **Пометки юриста** — textarea

**Правая панель — предпросмотр документа:**
- Тёмный фон (#1e293b)
- Toolbar сверху: имя файла + «стр. X из Y» + кнопки ◂ ▸ ⊕ ⊖
- PDF: встроенный рендер через iframe или pdf.js
- DOCX: отображать извлечённый текст (уже есть в базе после обработки)
- Белая «страница» по центру с тенью

**Стиль labels:**
- Секционные заголовки: uppercase, 10px, spacing 0.05em, цвет #94a3b8
- Field labels: обычный регистр (не КАПС), 10-11px, цвет #94a3b8
- Field values: 13px, цвет #1e293b

## Acceptance Criteria

1. Реестр: колонка «Уверенность» отсутствует в таблице
2. Реестр: bulk toolbar без кнопки Excel, кнопки с правильным шрифтом
3. Реестр: pagination footer полностью на русском
4. Реестр: боковая панель в Linear-style с секциями
5. Реестр: кнопка «Календарь» открывает календарь (не пустой экран)
6. Карточка: метаданные слева, превью справа на всю высоту
7. Карточка: PDF рендерится во встроенном preview
8. Карточка: DOCX показывает извлечённый текст
9. Карточка: «Открыть файл» открывает оригинал локально
10. Карточка: labels без КАПСА (кроме секционных заголовков)
11. Карточка: шаблон с двумя кнопками «Найти» + «Добавить»
12. Карточка: таймлайн версий с точками и ссылками «Сравнить»

## Constraints

- NiceGUI 3.x: CSS в @layer components / @layer overrides
- AG Grid: изменения через columnDefs
- Функциональные class names (status-active и т.д.) — не переименовывать
- Не менять backend логику — только UI
